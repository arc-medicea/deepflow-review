Mobile Experience Specification

DeepFlow on Mobile

Chat and voice first. A native mobile experience designed for operations managers checking status on the go, solo founders between meetings, and executives who need one-tap clarity.

01

Mobile Philosophy

Mobile DeepFlow isn't a shrunken desktop — it's a purpose-built command interface where chat and voice are the primary interaction model.

🎯 Core Decision
Chat-First, Not Dashboard-First
Ali asked: "What is the mobile experience? Chat and voice first?" — Yes. On mobile, the AI chat is the primary creation and control interface. Users speak or type natural language commands to create tasks, check status, and advance workflows. The dashboard exists for visual scanning, but chat is how you do things.

Default landing view: Chat. This is the AI-native choice. Ravi opens his phone between meetings and says "What needs attention today?" — DeepFlow responds with a summary. Sarah types "Mark Data Collection as complete" — done. James says "Approve the ICAAP submission" — confirmed. The dashboard is one tap away for visual scanning, but the chat is the universal remote.
📱 Triage, Not Create

Mobile users check status, advance tasks, and approve gates. Complex workflow creation happens on desktop. Mobile is optimised for fast reads and quick actions.

🗣️ Voice as First-Class

Between meetings, in a taxi, walking to lunch — voice input isn't an afterthought. The microphone button is always visible in chat, and AI reads back summaries aloud.

👆 One-Hand Friendly

Critical actions live in the bottom 60% of the screen — the thumb zone. The tab bar, chat input, and quick actions are all reachable with one thumb.

02

Mobile Navigation

A persistent bottom tab bar — not a hamburger menu. Five destinations, always visible, always one tap away. Inspired by the apps people already understand.

💡 UX Decision

Hamburger menus hide navigation and increase cognitive load. A bottom tab bar keeps all top-level destinations visible and within the thumb zone. Research consistently shows bottom tabs outperform hamburgers for engagement and discoverability (Nielsen Norman Group, 2019).

💡 Voice Design Decisions

Voice Input: Tap the microphone in chat to enter voice mode. Speech is transcribed in real-time and sent as a chat message. Great for Ravi between client meetings — "What's the status of the Q2 ICAAP project?" without touching the keyboard.

Voice Output: AI responses can be read aloud via text-to-speech. Toggle in settings or per-response. Particularly useful for status summaries — Ravi hears "3 tasks complete, 1 blocked, 1 awaiting your approval" while walking to his next meeting.

Conversation Context: Chat maintains context across the session. "Mark that as complete" refers to the last-mentioned task. "Assign it to Sarah" — DeepFlow knows what "it" is.

Chat Capabilities on Mobile

Create & Assign

"Set up a new ICAAP workflow for Q2" → DeepFlow generates the task graph and confirms

"Create a task: Review the Basel III amendments, assign to Molly, due Friday" → Task created with all parameters

Status & Summaries

"What needs attention?" → Summary of blocked, overdue, and awaiting-approval tasks

"How's the ICAAP project?" → Health overview with completion percentage and blockers

Advance & Control

"Mark Data Collection as complete" → Status updated, downstream tasks unblocked

"Approve the board submission" → Approval recorded with timestamp and attribution

Inline Task Previews

When the AI references a task, it renders as an interactive card within the chat bubble — showing status dot, task name, assignee, and due date. Tap to open the full task detail view.

💡 Dashboard Design Decisions

Information hierarchy: Greeting → Summary cards (2×2 grid) → Project health bars → Tasks needing attention. This mirrors Ravi's thought process: "Am I on track?" → "Which projects?" → "What specifically?"

Gesture & Interaction Reference

Swipe Right → Advance

Swipe a task row to the right to advance its status: Ready → In Progress → Complete. Green checkmark revealed behind the row. Release to confirm, or swipe back to cancel.

Swipe Left → Options

Swipe left reveals contextual actions: Reassign, Snooze, Flag. Colour-coded action buttons (blue, amber, red). Never more than 3 options — keep it quick.

Tap Status Symbol

Each status symbol is a 24px interactive target within a 44px touch area. Tap to advance: ● → ▶ → ✓. Long-press for status picker with all options.

Push Notification Strategy

What Triggers a Push

🔴 Immediate: Task blocked, AI execution failed, approval request
🟡 Timely: Task assigned to you, status change on watched task, due date approaching
🟢 Batched: AI task completions, team activity summaries (grouped hourly)

Push → Quick Action

iOS and Android support actionable notifications. "Board Approval needs your sign-off" shows two buttons: Approve and View. Approve works without opening the app — James taps from his lock screen and moves on.

07

Responsive Breakpoints

Three tiers — phone, tablet, desktop. Each tier isn't just smaller; it's a fundamentally different interaction model.

Breakpoint Width Layout Model Navigation
📱 Mobile < 640px Single-column, stacked views Bottom tab bar
📋 Tablet 640–1024px Two-column where useful (list + detail) Sidebar (collapsible) + tab bar optional
🖥️ Desktop > 1024px Three-panel workspace, graph canvas Top nav + sidebar

What Changes Per Breakpoint

Feature 📱 Mobile 📋 Tablet 🖥️ Desktop
Graph canvas ✗ Hidden ↔ Simplified ✓ Full
Task list ✓ Full-screen ✓ Left panel ✓ Embedded
Task detail ↔ Stacked (push view) ✓ Right panel ✓ Side panel
Kanban board ↔ Horizontal scroll ✓ Full ✓ Full
AI Chat ✓ Full-screen tab ↔ Floating panel ↔ Centre panel
Voice input ✓ Primary ✓ Available ↔ Available
Bottom tab bar ✓ Always visible ↔ Optional ✗ Not shown
Swipe gestures ✓ Primary ✓ Supported ✗ Not applicable
3D layer navigator ✗ Hidden ✗ Hidden ✓ Full
💡 Key Adaptation: Graph View

The graph canvas — DeepFlow's core differentiator — is not shown on mobile phones. Complex dependency graphs don't translate to 375px screens. Instead, mobile users interact with the graph through the task list (which surfaces dependency information) and the AI chat (which can answer "what depends on this?" or "what's blocking the report?"). On tablets, a simplified, read-only graph with pinch-to-zoom is available.

Mobile-Only Features

🗣️ Voice Input

While available on all platforms, voice is the star on mobile. The prominent microphone button and full-screen voice mode are mobile-first designs.

👆 Swipe Gestures

Swipe-to-advance and swipe-for-options are touch-native patterns that have no desktop equivalent. They're the fastest path on mobile.

🔔 Push Notifications

Lock-screen actionable notifications with Approve/View buttons. This is where James does 80% of his DeepFlow interactions.

Accessibility & Ergonomics

Touch targets, thumb zones, contrast ratios, and motion preferences. A professional tool must be usable by everyone.

Thumb Zone Map
Hard to reach
Stretch zone
Easy — Natural thumb arc

Primary actions (tab bar, chat input, task actions) live in the bottom 40% — the natural thumb arc for one-handed use.

Touch Target Rules

All interactive elements meet minimum sizing:

44 × 44px — Apple Human Interface Guidelines minimum

48 × 48px — Preferred for primary actions (tab bar items, CTA buttons)

8px minimum gap — Between adjacent touch targets to prevent mistouch

Status symbols: Visual size is 24px, but the tappable area extends to 44px with invisible padding

Task list rows: Full row is tappable (minimum 56px height), not just the text

Contrast & Readability

All text meets WCAG AA contrast ratios (4.5:1 for body text, 3:1 for large text). Status colours are never conveyed by colour alone — always paired with shape (dots, icons) and text labels.

In bright outdoor conditions (Ravi checking his phone outside a café), the light theme provides excellent readability. Dark mode available for preference and low-light.

Motion & Reduced Motion

All animations respect prefers-reduced-motion. When enabled:

• Voice waveform animation → static bars
• Swipe gestures → still functional, no spring/bounce
• Notification badge pulse → solid badge
• Page transitions → instant cut, no slide

Functionality is never gated behind animation.
