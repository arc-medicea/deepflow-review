Design Guidelines

The visual language for DeepFlow — an AI orchestration platform where clarity isn't optional, it's infrastructure. Monochrome base. Purposeful colour. Every pixel earns its place.

01

Brand Overview

What DeepFlow is, who it serves, and the single idea every design decision flows from.

Describe what you want. DeepFlow figures out how to get it done.

DeepFlow is an AI orchestration platform that transforms natural language into executable task graphs — dependency-aware, auditable, and governed. It bridges the gap between "I need this done" and the structured, accountable process that gets it done, coordinating humans and AI agents side by side.

🎯
Orchestration, Not Automation

DeepFlow doesn't replace people — it coordinates them. Humans and AI work the same graph, with clear handoffs, dependencies, and accountability.

🔍
Governed Transparency

Every action is timestamped, attributed, and auditable. Built for industries where "who did what and when" isn't optional — it's regulation.

⚡
Power Without Complexity

Enterprise capability, approachable interface. The user describes intent in plain language; the platform handles decomposition, sequencing, and execution.

02

Design Principles

Five rules that every design decision must pass through. When in doubt, refer here.

01
Calm by Default, Vivid by Exception

The interface is monochrome. Colour is reserved for information that demands attention: status changes, actionable items, brand moments. If everything is colourful, nothing is. Restrained palette = stronger signal.

In practice → A graph of 50 nodes is rendered in neutral grays. The 3 nodes that need attention glow with status colour. Your eye goes exactly where it should.
02
Show the Structure, Hide the Complexity

DeepFlow manages intricate dependency graphs. The UI's job is to make that structure legible — not to expose every node, edge, and metadata field at once. Progressive disclosure: show what matters now, reveal depth on demand.

In practice → A collapsed parent task shows its name, status, and child count. Expanding it reveals children in a shaded containment area. You drill down when you choose to, not because the UI forced everything on screen.
03
Status Is Sacred

In an orchestration platform, status is the most important piece of information. It must be visible, consistent, and unambiguous in every view — graph, kanban, list, dashboard. One visual language for status, everywhere.

In practice → Green means complete. Red means failed. Amber means needs attention. Always. In every view, at every zoom level. No exceptions, no "it depends."
04
Readable Before Beautiful

Users managing 50+ tasks across multiple workstreams cannot afford to squint. Task names are never truncated to invisibility. Labels are always present alongside icons. Contrast meets WCAG AA minimum. Legibility is non-negotiable.

In practice → If a task name doesn't fit, the container grows — not the text shrinks. Icon-only buttons always have a tooltip, and primary actions always have visible text labels.
05
Human and AI, Visually Distinct

DeepFlow uniquely blends human and AI task execution. At a glance, users must always know whether a task is assigned to a person or an agent. This distinction is expressed through shape, never through colour (colour is reserved for status and brand).

In practice → Circular avatars = human assignees. Hexagonal avatars = AI agents. This shape language persists across graph nodes, list rows, kanban cards, and detail panels.

Voice Principles

Clarity Over Cleverness

Users are managing regulated processes. Use plain language. Say "3 tasks are blocked" not "Some tasks need a nudge." Save personality for empty states and onboarding — never in error messages.

Action-Oriented

Tell the user what to do, not just what happened. "Task failed — retry or reassign" is better than "Task failed." Every message should imply a next step.

Respect the User's Time

No filler. No "Please note that..." or "It appears that..." Get to the point. Sarah has 50 tasks to manage — every word must earn its place.

Colour Palette

Monochrome + pops of colour. The canvas is calm; colour earns its place by meaning something.

The Philosophy

The interface is built on a monochrome neutral scale — dark backgrounds, light text, subtle borders. Colour appears only for three purposes: brand identity (cyan/blue gradient), semantic status (success, warning, error), and parent-child grouping (subtle containment shading). If a colour doesn't serve one of these three roles, it doesn't belong.

Brand — Cyan/Blue Gradient

The DeepFlow signature. Used for primary actions, interactive elements, links, brand moments, and the hero gradient. This is the only "decorative" colour in the system.

Grouping — Parent-Child Containment

When parent tasks are expanded to show children, a subtle tinted background groups them visually. Nesting levels use progressively lighter washes. This replaces the depth-colour rainbow with something calmer and more scalable.

▾ L1 — ICAAP Document Production
Data Collection Complete
Risk Assessment In Progress
▾ L2 — Capital Modelling
Run Capital Model Complete
Gather Market Data Needs Attention
Draft Report Pending

Containment uses the brand cyan at very low opacity (6% for L1, 3% for L2) — enough to perceive grouping without competing with status colours. Borders use slightly higher opacity for definition.

Usage Rules

Hierarchy Through Weight

Use weight (not size) to create subtle hierarchy within the same size level. Semibold (600) for labels, Regular (400) for values. Bold (700) only for headings. Extra Bold (800) only for display/hero text.

Negative Letter-Spacing for Large Text

Text above 24px uses letter-spacing: -0.02em. Display text uses -0.03em. Body text uses default spacing. This keeps large headings tight and readable.

Monospace for Data

Use SF Mono / Fira Code for hex values, timestamps, IDs, and code references. This is never a primary typeface — only for technical metadata.

No Truncation Without Tooltip

If text must be truncated (and it should be rare), a tooltip showing the full text is mandatory. Task names should expand their containers before resorting to truncation.

Iconography

Clean, consistent, and always paired with a label. Icons support comprehension — they never replace it.

Style: Outlined

1.5px stroke weight, rounded caps and joins. Consistent with Inter's geometric character. We recommend Lucide as the icon set — open source, React-native, and visually aligned with our aesthetic.

Sizes

Labels Are Mandatory

Every icon button has a visible text label. The only exception is universally recognised icons (close ✕, search 🔍) in contexts where space is critically constrained — and even these must have tooltips.

09

Elevation & Depth

Three tiers of elevation create spatial hierarchy. In dark mode, elevation is expressed through background lightness and shadow — higher = lighter + more shadow.

Flat
Cards, list items, containers

Background: neutral-900
Border: 1px neutral-800
Shadow: none

Raised
Hover states, dropdowns, tooltips

Background: neutral-850
Border: none
Shadow: shadow-md

Floating
Modals, command palette, overlays

Background: neutral-800
Border: none
Shadow: shadow-lg

Elevation Rules

In dark interfaces, shadow alone isn't enough to convey elevation — the eye primarily reads "lighter = closer." Combine background lightness with shadow intensity. Never use more than 3 elevation levels simultaneously in a single view. The page background (neutral-950) is the ground plane; everything sits above it.

10

Motion & Animation

Motion is functional, not decorative. It communicates state changes, guides attention, and provides spatial continuity. Hover over the cards to preview timing.

Fast — 150ms
cubic-bezier(0.4, 0, 0.2, 1)
Hover states, toggles, small reveals
Normal — 250ms
cubic-bezier(0.4, 0, 0.2, 1)
Panel slides, expand/collapse, status changes
Slow — 400ms
cubic-bezier(0.4, 0, 0.2, 1)
Page transitions, modal enter/exit, graph layout shifts
Motion Principles

Purposeful: Every animation answers "what changed?"
Quick: Default to fast. Only use slow for major transitions.
Respectful: Honour prefers-reduced-motion — reduce to opacity-only fades or disable entirely.
Consistent: One easing curve for everything. No bouncing, no spring physics.

Graph-Specific Motion

Edge flow: Subtle animated dots along dependency edges indicating data/status flow direction. Speed proportional to zoom level.
Node entrance: Fade + slight scale (0.95→1.0) when nodes appear, staggered by 30ms per node.
Status change: Brief pulse glow on the node when status updates (150ms).
