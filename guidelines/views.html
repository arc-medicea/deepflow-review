Layout Specifications

Top Bar — Fixed at 48px height. Contains breadcrumb navigation (click any level to navigate up), view switcher (Graph / Kanban / List — same data, different presentation), and filter controls. The Chat button provides a secondary entry point to the AI panel.
Left Sidebar — Collapsed: 56px (icon-only navigation). Expanded: 240px (full project tree). Toggle via hamburger icon. Contains workstream navigation with status dots. Sarah keeps this collapsed; Ravi uses it expanded to jump between client projects.
Graph Canvas — Fills remaining space. Dot-grid background provides spatial anchoring. Contains the dependency graph with nodes, edges, parent containment zones, minimap (bottom-right, 160×100px), and zoom controls (bottom-left). Canvas pans and zooms with standard gestures.
Right Detail Panel — 360px width. Slides in when a node is selected, slides out on close. Tabbed interface: Info, Resources, History, Sub-tasks. Shows dependencies in both directions — critical for Sarah's "what's blocking this?" workflow.
Parent-Child Containment — L1 groups use rgba(0, 119, 221, 0.05) background with dashed border at 0.12 opacity. L2 nesting uses 0.03 background — lighter to maintain visual hierarchy. Labels sit on the top edge, anchored to the group.

Decision

Where Does Chat Live?

Chat is DeepFlow's primary AI interaction surface — how users create, modify, and query workflows using natural language. It must be prominent without consuming graph space.

Recommendation: Collapsible Overlay Panel

Chat occupies a 380px overlay panel that slides in from the right edge of the graph canvas, not the viewport. When open, it overlays the graph — the graph does not shrink. This is deliberate:

  • No layout shift. Graph nodes don't jump when chat opens/closes. Sarah's mental model of her graph stays stable.
  • First-class but not permanent. The branded edge tab (gradient, always visible) signals "AI is here" without consuming space. One click to open.
  • Context-aware suggestions. AI sees what you see — it references the current graph state and offers contextual actions.
  • Non-competing with detail panel. Detail panel is on the layout grid (pushes graph). Chat is an overlay. Both can be "conceptually open" — detail panel shows selected node info while chat discusses the broader workflow.

Alternatives Considered

Option Pros Why Not
Floating bottom-right widget Familiar pattern (Intercom-style). Minimal footprint. Too small for workflow creation. Feels like "support chat", not a first-class feature. Truncates AI responses.
Graph shrinks to accommodate chat Chat always visible. No overlap. Layout shift breaks spatial memory. On narrow screens, graph becomes unusably small. Conflicting resize with detail panel.
Bottom drawer (horizontal) Doesn't compete horizontally with detail panel. Vertical chat is more natural for conversation. Bottom drawers feel like developer tools. Reduces graph height — worse for shallow-wide DAGs.
✓ Right-edge overlay panel Full-height for long conversations. Prominent but dismissible. No layout shift. Branded entry point always visible. —
Mutual exclusivity note: When chat panel overlays the graph AND the detail panel is open, the graph visible area narrows significantly. On screens < 1440px, opening chat should auto-close the detail panel (with a toast: "Detail panel minimised"). On wider screens, both can coexist — the graph still has ~400px+ of visible canvas.

Layout Specifications

Header — Personalised greeting with contextual summary stats. The "New Workflow" gradient CTA is the single primary action per the Rule of One. Filter buttons (project status, date range) use ghost style. On mobile, filters collapse into a dropdown.
Attention Banner — Appears only when items need intervention. Uses warning semantic colour. Answers "what needs me right now?" before the user has to scan. Dismissable once acknowledged.
Project Cards — 2-column grid on desktop, 1-column on mobile. Each card contains: project name + client, 5-segment health meter, progress bar with percentage and due date, task status counts (colour-coded dots), assignee avatar stack (circles for humans, hexagons for AI), and last activity timestamp.
Health Meter — 5 segments. Green = on track, Amber = at risk, Empty = poor. Calculated from: % completion vs. expected, overdue tasks, failed tasks. More intuitive than a single number — Ravi can scan 10 projects in seconds.
Status Counts — The coloured dot + number chips use the same status colours from the guidelines (success/info/warning/error/pending). Ravi sees "1 red" and knows something failed without reading a word.

Layout Specifications

Search — Full-width search input with focus state (brand-solid border). Filters for status (Active / Complete / At Risk / All) and team. Search is fuzzy — matches project name, owner name, and tags.
Template Section — Horizontally scrollable card row. Each template card shows a miniature graph preview (abstract dot-and-line representation of the workflow structure), template name, and complexity indicator (task count + stage count). The "Blank Project" card uses dashed border to indicate creation.
Project Cards — 3-column grid on desktop, 2-column on tablet, 1-column on mobile. Each card has a graph thumbnail preview area (tinted by project health status), project name, owner (with avatar), and status badge. The thumbnail uses the same miniature dot representation from templates — giving visual distinction between projects with different graph structures.
Graph Thumbnails — Not full graph renders. They're abstract representations: coloured dots at approximate positions, tinted by status. This gives a visual "fingerprint" per project — enough to recognise a familiar workflow at a glance without rendering the full ReactFlow graph at thumbnail scale (which would be illegible).

Breakpoint Graph View Workspace View Gallery View
Desktop
≥ 1440px		 Full layout: sidebar + canvas + detail panel. Chat overlays graph. All panels can coexist. 2-column project grid. Full header with greeting and filters. 3-column gallery grid. Template row fully visible.
Laptop
1024–1439px		 Sidebar starts collapsed. Opening chat auto-closes detail panel. Canvas dominates. 2-column project grid. Filters may wrap to second line. 2-column gallery grid. Template row scrolls horizontally.
Tablet
768–1023px		 Sidebar hidden (accessible via hamburger). Detail panel becomes full-screen overlay. Chat is full-screen modal. 1-column project cards. Attention banner wraps. 2-column gallery. Templates collapse to horizontal scroll.
Mobile
< 768px		 No graph canvas. Defaults to List View. Task detail as bottom sheet. Chat as full-screen modal. Graph is a desktop feature. 1-column cards. Filters in dropdown. Greeting simplified. 1-column gallery. Templates as swipeable cards.
Key Decision: No Graph on Mobile. Dependency graphs with 20+ nodes are illegible at 375px. Rather than deliver a degraded experience, mobile defaults to List View — which shows the same data (tasks, status, assignees, dependencies as indented rows) in a mobile-appropriate format. Users can still browse, update status, and chat with AI. Graph editing is a desktop activity.
Ravi on Mobile: Checks Workspace View between meetings (1-column cards with progress bars). Taps into a project for List View. Uses chat to quick-update a task. He doesn't need the graph on his phone — he needs to know what's on fire and handle it.
